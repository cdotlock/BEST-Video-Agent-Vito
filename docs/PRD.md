# PRD：BEST Video Agent 通用化与自进化升级

- 版本：v0.3
- 日期：2026-03-04
- 状态：Ready for Build
- 范围：产品能力、架构边界、实施验收

## 1. 产品定位

将当前偏“单一路径小说工作流”的系统，升级为**通用视频创作 Agent 工作台**：

1. 面向任意题材与任意画风，不预设单一风格。
2. 通过 `Skills + MCP + Memory` 原子化组织能力，不依赖外部 prompt 平台。
3. 支持多路径流程编排、自动回顾与策略进化。

## 2. 核心目标

## 2.1 Must Have

1. 生成前初始化（风格确认）成为默认入口。
2. 支持风格检索 + 参考图反推 + 风格档案固化。
3. 支持多条生产路径并可按任务动态切换。
4. 支持多段视频生成与拼接计划。
5. 系统长期记忆默认开启，越用越贴近用户偏好。
6. 执行模式支持：
   - `checkpoint`：关键节点确认
   - `yolo`：自动推进，不等待确认

## 2.2 Non-goal（本阶段不做）

1. 不重写 Agent loop 协议。
2. 不引入新的私有协议，保持 MCP 标准。
3. 不做重型 NLE（复杂轨道/关键帧剪辑）。

## 3. 设计原则

1. Agent Native：决策由 Agent 驱动，工具只做原子能力。
2. Prompt 内置化：prompt 优化完全依赖内置 skills + memory。
3. 流程可解释：每次路径选择都可回溯理由。
4. 结果可复用：中间产物都持久化，防止长对话丢失。
5. 默认简单：高频路径直达，低频能力收纳到二级层。

## 4. 目标能力模型

## 4.1 工作流路径图（Path Graph）

系统内置多条可组合路径，Agent 可按上下文选择：

1. `path.style_bootstrap`
   - 公共图库搜图 -> 反推 -> 生成 StyleProfile
2. `path.storyboard_density`
   - 单图分镜 / 四宫格 / 九宫格
3. `path.image_to_video.first_frame`
   - 首帧图生视频
4. `path.image_to_video.first_last_frame`
   - 首尾帧约束图生视频
5. `path.multi_reference_video`
   - 图 + 视频参考混合驱动视频生成
6. `path.multi_clip_compose`
   - 多条视频候选 -> 拼接计划 -> 导出清单

每条路径包含：

1. 触发条件
2. 必要输入
3. 工具序列
4. 失败回退路径
5. 评审指标

## 4.2 参考信息角色化

参考素材不再只有“参考图”一种语义，统一角色化：

1. `style_ref`：画风参考
2. `scene_ref`：场景构图参考
3. `empty_shot_ref`：空镜氛围参考
4. `character_ref`：角色立绘参考
5. `motion_ref`：视频运动参考

Agent 在组装 prompt 时按角色注入，不再把所有参考混在一起。

## 4.3 分镜输出密度

分镜生成支持三种密度：

1. `single`：单张分镜（默认）
2. `grid_2x2`：四宫格
3. `grid_3x3`：九宫格

网格分镜为中间资产，可直接用于后续图生视频路径。

## 4.4 视频生成策略

视频生成支持策略化输入：

1. `first_frame`：首帧 + prompt
2. `first_last_frame`：首尾帧 + prompt
3. `mixed_refs`：图/视频多参考 + prompt

输出结果至少包括：

1. 任务参数快照
2. 结果 URL（或任务 ID）
3. 可追溯的 prompt 与参考列表

## 4.5 多段拼接与最简剪辑

内置最简剪辑能力（非重型编辑器）：

1. 片段排序
2. 片段时长裁切（入点/出点）
3. 片段转场类型（none/cut/fade）
4. 导出拼接计划 JSON（供后端执行或外部工具消费）

## 5. 自进化闭环

## 5.1 事件采集

采集以下行为并结构化入库：

1. 路径选择与切换
2. 参考素材采纳/否定
3. prompt 优化前后对比
4. 片段保留/丢弃
5. 最终导出选择

## 5.2 记忆提炼

将行为提炼为偏好：

1. 风格 token 偏好
2. 参考角色偏好
3. 路径偏好（如用户偏爱 `first_last_frame`）
4. 剪辑偏好（节奏、转场）

## 5.3 策略推荐

下次任务默认注入：

1. 路径推荐（Top-N）
2. 默认参考角色权重
3. 默认 prompt 约束片段

## 5.4 Agent Review

每个关键阶段执行自评审：

1. 当前路径是否仍最优
2. 是否需要增补分镜密度
3. 是否需要改为首尾帧策略
4. 是否应生成多候选视频用于拼接

评审结论写回长期记忆。

## 6. 能力原子化（Skills + MCP）

## 6.1 内置 Skills（产品路径默认）

1. `video-mgr`
2. `style-search`
3. `video-memory`
4. `upload`

默认不在用户路径暴露低频技能（如动态 MCP 构建等）。

## 6.2 MCP 能力分层

1. 风格层：搜索、反推、档案应用
2. 生成层：图片生成、视频生成、网格分镜生成
3. 策略层：路径推荐、路径评审写回
4. 拼接层：片段组合计划

## 6.3 Prompt 管理原则

1. 不依赖 Langfuse 等外部 prompt 运行时服务。
2. 可参考 `docs/langfuse-baseline/` 的历史快照进行语义迁移。
3. 运行时 prompt 由 `Style Layer + Memory Layer + Task Layer` 动态组装。

## 7. 数据与接口约束

## 7.1 数据原则

1. 继续复用 `domain_resources` 作为统一素材表。
2. 新增字段优先，避免破坏性迁移。
3. 所有关键中间产物必须持久化。

## 7.2 API/MCP 约束

1. API routes：Zod 校验 + 调 service。
2. MCP providers：tool schema + Zod 校验 + 调 service。
3. 业务逻辑统一放 `src/lib/services/`。

## 8. UI/UX 要求（与 UIUX.md 对齐）

1. 白色主视觉、简洁高效、二级菜单明确。
2. 新手 4 步引导为默认入口。
3. 素材展示与操作直观，减少术语负担。
4. 最简剪辑能力纳入工作台，不跳出系统。

## 9. 里程碑与验收

## Phase A（能力）

1. A1 文档完成（本版）。
2. A2 原子能力落地（路径/分镜网格/多参考/拼接方案）。
3. A3 长期记忆进化落地（路径推荐 + 评审写回）。
4. A4 Playbook/useCase 更新。

## Phase B（体验）

1. B1 UIUX 规格完成。
2. B2/B3/B4 页面重构、新手引导、最简剪辑落地。
3. B5 README 与操作说明同步。

## Phase C（去外部依赖）

1. 清理 Langfuse 运行时依赖。
2. 内置 prompt 优化与记录闭环。
3. 保留一次性基线快照供迁移参考。

## 10. 风险与缓解

1. 路径过多导致混乱：
   - 用路径推荐 + 可解释理由，默认只显示推荐路径。
2. 记忆污染：
   - 只写入“明确行为”触发事件，支持按用户清空。
3. UI 重构影响旧用户：
   - 保留渐进披露与可回退入口，不直接暴露全部高级能力。
