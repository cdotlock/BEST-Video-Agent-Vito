# UIUX 设计规格（白色高效工作台）

- 版本：v1.1
- 日期：2026-03-04
- 状态：Ready for Build
- 目标页面：`/video`、`/video/[projectId]`
- 组件体系：基于 [Ant Design 组件总览](https://ant.design/components/overview-cn/)

## 1. 体验目标

用户进入系统后，30 秒内回答三个问题：

1. 我在哪个项目和序列？
2. 下一步该做什么？
3. 结果在哪看、怎么继续？

并在 5 分钟内完成：

1. 首次风格初始化
2. 第一轮分镜/视频产出
3. 一次最简拼接计划保存

## 2. 设计原则

1. 白色主视觉：浅底 + 清晰层级，不回退深色工程台风格。
2. 心流优先：每一屏只聚焦当前任务，降低操作分叉。
3. 渐进披露：默认只展示高频能力，低频高级能力放二级菜单。
4. 可控自动化：`checkpoint` 与 `yolo` 差异始终可见。
5. 一致反馈：每个异步动作都有进度、结果、下一步提示。

## 3. 信息架构

## 3.1 `/video`（项目入口）

主结构：

1. 顶部：品牌 + 全局状态 + 快速入口
2. 主体左：项目列表（搜索/排序/筛选）
3. 主体右：新手引导卡 + 快速创建

核心动作：

1. 创建项目
2. 查看最近项目
3. 进入项目工作台

## 3.2 `/video/[projectId]`（工作台）

三栏结构（桌面）：

1. 左栏（导航与上下文）
   - 项目与序列
   - 会话
   - 新手步骤状态
2. 中栏（创作主舞台）
   - Chat / Timeline / Storyboard / Clip 四视图
3. 右栏（素材与属性）
   - 分类素材
   - 筛选器
   - 快捷动作

移动端：

1. 左右栏改 Drawer
2. 中栏保持单列主舞台
3. 保留同样的四视图切换

## 4. 导航与菜单

## 4.1 一级导航

1. 项目
2. 创作
3. 素材
4. 风格
5. 历史

## 4.2 二级菜单规则

1. 仅展开当前一级节点。
2. 每个二级菜单最多 6 项，避免滚动式菜单噪音。
3. 工程能力（MCP 管理、技能调试）不进入默认产品路径。

## 5. 视觉系统

## 5.1 色板与 Token

1. Page BG：`#F5F7FA`
2. Card BG：`#FFFFFF`
3. Border：`#E5E7EB`
4. Text Primary：`#111827`
5. Text Secondary：`#6B7280`
6. Brand：`#2563EB`
7. Success：`#10B981`
8. Warning：`#F59E0B`
9. Error：`#EF4444`

## 5.2 组件映射（Ant Design）

1. 布局：`Layout`, `Sider`, `Drawer`, `Splitter`
2. 导航：`Menu`, `Segmented`, `Tabs`, `Breadcrumb`
3. 表单：`Form`, `Input`, `Select`, `Upload`, `Switch`
4. 数据：`Card`, `Collapse`, `List`, `Table`, `Timeline`
5. 反馈：`Alert`, `Message`, `Modal`, `Spin`, `Progress`

## 5.3 动效策略

1. 首屏加载：分区渐进出现（不超过 250ms）
2. 素材新增：卡片高亮闪烁一次
3. 路径切换：主视图平滑切换，不用复杂动画

## 6. 关键用户心流

## 6.1 新手首轮心流（默认）

1. 创建项目
2. 上传序列
3. 风格初始化（搜图 -> 反推 -> 应用）
4. 选择模式（Checkpoint/YOLO）
5. 生成内容
6. 在 Storyboard 查看结果
7. 在 Clip 视图保存拼接计划

## 6.2 老手复用心流

1. 进入项目
2. 复用历史风格档案
3. 直接发起批量生成
4. 快速筛选与拼接

## 7. 新手引导规格

## 7.1 首次引导条（固定顶部）

四步状态条：

1. 创建项目
2. 上传序列
3. 风格确认
4. 开始生成

每步包含：

1. 一句说明
2. 一个主按钮
3. 状态（未开始/进行中/已完成）

## 7.2 空态文案

1. 无项目：`还没有项目，先创建一个。`
2. 无序列：`先上传序列文本开始创作。`
3. 无风格：`先选参考图，系统会自动反推风格。`
4. 无素材：`还没有结果，点击“开始生成”。`

## 7.3 错误文案

必须包含：

1. 出错原因（简短）
2. 建议动作（立即可执行）

## 8. 核心交互细节

## 8.1 模式切换（Checkpoint/YOLO）

1. 顶栏和输入区双重显示当前模式。
2. 切换到 YOLO 时弹一次提示。
3. 新用户默认 `checkpoint`。

## 8.2 Style Init 抽屉

结构：

1. 搜索区
2. 参考图区
3. 反推结果区
4. 已保存 profile 区

动作：

1. 一键应用到当前序列
2. 一键把应用结果注入聊天继续执行

## 8.3 Storyboard 视图

支持：

1. 分类分组
2. 素材预览
3. 快捷动作：
   - 复制 prompt
   - 继续生成
   - 设为风格参考

## 8.4 Clip（最简剪辑）视图

目标：快速形成拼接计划，不做复杂时间线编辑。

功能：

1. 选择视频片段
2. 拖拽排序
3. 每片段设置：
   - in/out（秒）
   - 转场（none/cut/fade）
4. 保存拼接计划（JSON）

## 9. 可用性指标与验收

## 9.1 UX 指标

1. 首次用户 5 分钟内完成首轮生成。
2. 主流程点击步数较现版下降 30%。
3. 空态页面不出现“无指导空白区”。

## 9.2 视觉验收

1. 默认全部主页面为白色体系。
2. 深色仅允许在媒体预览内容中出现，不作为容器基底。
3. 组件圆角、边框、阴影风格一致。

## 9.3 功能验收

1. 新手引导四步完整可用。
2. `checkpoint/yolo` 行为差异清晰可见。
3. Storyboard 与 Clip 视图均可完成闭环操作。
4. 移动端可完成完整主流程。

## 10. 实施顺序（开发执行）

1. 统一主题 token 与布局骨架
2. 首次引导与空态
3. Style Init 与 Storyboard 优化
4. Clip 视图与拼接计划
5. 细节交互和移动端修正

## 11. 实现态映射（v1.1）

## 11.1 `/video` 首页

1. 左侧主区：
   - 快速创建卡片（输入项目名 -> 立即创建并进入）
   - 项目列表（搜索、打开、删除）
2. 右侧辅区：
   - 工作台状态（总项目数、最近更新项目）
   - 新手 4 步说明（从创建到 clip 保存）
3. 视觉要求：
   - 页面底色固定 `#F5F7FA`
   - 所有容器白底卡片 + 统一边框/圆角

## 11.2 `/video/[projectId]` 工作台

1. 顶栏：
   - Breadcrumb（Projects / 当前项目）
   - 视图切换（Chat/Timeline/Storyboard/Clip）
   - 模式切换（Checkpoint / YOLO）
2. 左栏：
   - 一级/二级菜单（创作、风格、素材、历史）
   - 序列/会话管理
3. 中栏：
   - 新手引导卡（4 步 + 一键动作）
   - 状态提醒（无序列、无风格 profile）
   - Style Init 入口
   - 主舞台（Chat/Timeline/Storyboard/Clip）
4. 右栏：
   - 素材面板（搜索、媒体类型筛选、分组展示）
5. 移动端：
   - 左右栏全部 Drawer 化，保留同等功能

## 11.3 Style Init（抽屉）

1. 搜图区：公共图库搜索 + 来源状态回显。
2. 反推区：profile 命名、creative goal、正负 prompt 回显。
3. 记忆区：
   - 长期记忆默认建议（style token/provider/query）
   - 路径推荐卡（Adopt/Skip）
4. 保存区：历史 profile 快速复用并注入到 chat。

## 11.4 Clip Composer（最简剪辑）

1. 功能闭环：
   - 选片段
   - 排序（上移/下移）
   - in/out 设置
   - 转场设置（none/cut/fade）
   - 保存 plan JSON
2. 辅助能力：
   - 一键加入全部视频素材
   - 总时长实时统计
   - 清晰空态与错误提示

## 12. 引导文案规范

所有引导文案必须满足：

1. 指向明确动作（例如“去初始化”“去上传”）。
2. 不出现内部术语（如 tool registry、provider catalog）。
3. 同时说明“为什么做”和“做完会得到什么”。
4. 不超过 2 句，避免段落化说明。

## 13. 组件约束（Ant Design）

1. 菜单与导航只用 `Menu + Segmented + Breadcrumb` 组合。
2. 表单交互只用 `Input/Select/Button/Switch/Upload`，不引入自定义 form 框架。
3. 状态反馈只用 `Alert/Tag/Empty/Spin/Message`，禁止隐式静默失败。
4. 卡片布局优先 `Card`，列表优先 `List`，避免自制复杂容器样式。
