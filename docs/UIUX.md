# UIUX 重构文档（用户视角版）

- 版本：v0.2
- 日期：2026-03-04
- 状态：Ready for Build
- 适用范围：`/video`、`/video/[projectId]`
- 依赖设计体系：[Ant Design 组件总览](https://ant.design/components/overview-cn/)

## 1. 先回答当前问题

本次文档先明确你提出的 4 个问题，并给出可执行改造方向。

1. 问题：UI 没有完成白色重构。
2. 问题：页面语义和布局让人看不懂。
3. 问题：缺少以用户任务为中心的 UX 设计。
4. 问题：没有新手引导，首次使用成本过高。

本版 UIUX 的目标是：

1. 默认白色主视觉，不再默认深色工程风。
2. 一屏回答用户三个问题：我在哪、下一步做什么、结果在哪里看。
3. 以“新手 5 分钟产出第一条可用分镜”为核心任务流。
4. 在不破坏 Agent Native 架构的前提下，降低认知门槛。

## 2. 设计原则（严格执行）

1. 白色优先：默认浅色主题，深色不作为默认入口。
2. 任务优先：所有主按钮围绕“创建项目、上传素材、确认风格、开始生成、查看结果”。
3. 术语克制：用户侧只暴露必要概念，隐藏低频工程概念。
4. 渐进披露：基础路径先跑通，高级能力放入二级菜单。
5. 可回退：关键破坏操作必须二次确认，保留可恢复路径。

## 3. 信息架构（重排）

## 3.1 一级结构

1. `项目`：项目与序列管理。
2. `创作`：聊天生成与执行模式。
3. `素材`：图片/视频/JSON 结果查看与复用。
4. `风格`：公共图库搜索、反推、风格档案管理。
5. `历史`：会话与关键事件时间线。

## 3.2 二级菜单规则

1. 默认只展开当前一级菜单的二级项。
2. 二级项只放“当前任务相关”的过滤和快捷动作。
3. 工程能力（MCP、Skills 调试）不进入默认用户菜单。

## 4. 视觉规范（白色主视觉）

## 4.1 色彩 Token

1. 页面底色：`#F5F7FA`。
2. 卡片底色：`#FFFFFF`。
3. 边框：`#E5E7EB`。
4. 主文本：`#111827`。
5. 次文本：`#6B7280`。
6. 交互主色：`#2563EB`。
7. 成功：`#10B981`。
8. 警告：`#F59E0B`。
9. 错误：`#EF4444`。

## 4.2 组件策略（Ant Design）

1. 使用 `ConfigProvider` 统一主题 token。
2. 导航层：`Menu`、`Tabs`、`Dropdown`、`Breadcrumb`。
3. 表单层：`Form`、`Input`、`Select`、`Upload`、`Switch`。
4. 展示层：`Card`、`Collapse`、`Image`、`Tag`、`Timeline`。
5. 反馈层：`Alert`、`Drawer`、`Modal`、`Message`、`Spin`。
6. 禁止同一页面混用多套视觉风格。

## 5. 首次使用引导（必须落地）

## 5.1 First-Run 引导条

首次进入 `/video` 时，顶部显示 4 步引导条：

1. 创建项目。
2. 上传序列文本。
3. 选择风格参考图。
4. 开始生成。

每一步必须有：

1. 一句人话说明。
2. 一个主操作按钮。
3. 完成状态标记。

## 5.2 空态文案规范

1. 项目为空：`还没有项目。先创建你的第一个视频项目。`
2. 序列为空：`还没有序列。上传一个 .md 或 .txt 开始。`
3. 风格未初始化：`先选参考图，系统会自动反推风格提示词。`
4. 素材为空：`还没有生成结果。点击“开始生成”获取第一批素材。`

## 5.3 新手保护

1. 未选择序列时，禁用发送区并给出明确提示。
2. YOLO 模式切换时显示提醒：`YOLO 将自动推进，不再逐步确认`。
3. Checkpoint 模式默认开启，新用户先走可控路径。

## 6. 核心工作流（用户任务流）

## 6.1 主流程

1. 新建项目。
2. 上传序列文件。
3. 打开风格初始化面板。
4. 搜索公共图库图片。
5. 勾选参考图并执行反推。
6. 应用风格档案到当前序列。
7. 发送生成指令（Checkpoint 或 YOLO）。
8. 在素材区和 Storyboard 中复用结果继续迭代。

## 6.2 执行模式 UX

1. Checkpoint：关键节点弹确认，不自动推进下一阶段。
2. YOLO：完成初始化后直接持续推进，不等待确认。
3. 模式状态在顶部与输入区双处可见，避免“当前模式不明确”。

## 7. 页面级重构要求

## 7.1 `/video` 项目页

1. 强化“创建项目”输入与主按钮。
2. 项目卡片展示名称、更新时间、最近状态。
3. 删除操作放二级操作并二次确认。

## 7.2 `/video/[projectId]` 工作台

1. 左侧固定“序列 + 会话”。
2. 中间固定“聊天/时间线/分镜”三视图切换。
3. 右侧固定“素材分组 + 筛选 + 快捷动作”。
4. 顶部固定“模式开关 + 刷新 + 全局状态”。

## 7.3 移动端

1. 左右栏改抽屉。
2. 顶部保留当前步骤与主操作。
3. 保证上传、发送、查看素材三条路径不超过两次点击。

## 8. 交互一致性规范

1. 按钮命名统一动词开头：`创建`、`上传`、`应用`、`生成`、`继续`。
2. 不在同层混用技术术语与业务术语。
3. 错误提示必须包含“下一步建议”。
4. 任何耗时动作都显示进行状态。

## 9. 验收清单（UIUX 完成标准）

以下 10 条全部满足，才算 UIUX 重构完成：

1. 默认进入页面为白色主视觉。
2. 新手首次进入能看到清晰 4 步引导。
3. 用户无需阅读文档即可完成第一轮生成。
4. Checkpoint / YOLO 差异有明确提示。
5. 素材区支持分类筛选和快捷复用。
6. 页面术语对新手可理解，无工程黑话堆叠。
7. 空态文案不再出现“无上下文空白区”。
8. 移动端可完成完整主流程。
9. 高级能力不干扰默认路径。
10. 视觉层级一致，无深色遗留主区域。

## 10. 实施顺序（按风险最小化）

1. 先做白色主题与布局层。
2. 再做新手引导与空态文案。
3. 再做风格初始化流程的引导强化。
4. 最后做二级菜单与快捷操作收敛。
