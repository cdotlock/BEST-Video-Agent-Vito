# UIUX 设计规格（白色高效工作台）

- 版本：v1.4.2
- 日期：2026-03-05
- 状态：Ready for Build
- 目标页面：`/video`、`/video/[projectId]`
- 组件体系：基于 [Ant Design 组件总览](https://ant.design/components/overview-cn/)

## 1. 体验目标

用户进入系统后，30 秒内回答三个问题：

1. 我在哪个项目和序列？
2. 下一步该做什么？
3. 结果在哪看、怎么继续？

并在 5 分钟内完成：

1. 首次风格初始化
2. 第一轮分镜/视频产出
3. 一次最简拼接计划保存

## 2. 设计原则

1. 白色主视觉：浅底 + 清晰层级，不回退深色工程台风格。
2. 心流优先：每一屏只聚焦当前任务，降低操作分叉。
3. 渐进披露：默认只展示高频能力，低频高级能力放二级菜单。
4. 可控自动化：`checkpoint` 与 `yolo` 差异始终可见。
5. 一致反馈：每个异步动作都有进度、结果、下一步提示。

## 3. 信息架构

## 3.1 `/video`（项目入口）

主结构：

1. 顶部：品牌 + 全局状态 + 快速入口
2. 主体左：项目列表（搜索/排序/筛选）
3. 主体右：简洁状态卡 + 快速创建

核心动作：

1. 创建项目
2. 查看最近项目
3. 进入项目工作台

## 3.2 `/video/[projectId]`（工作台）

三栏结构（桌面）：

1. 左栏（导航与上下文）
   - 项目与序列
   - 会话
   - 计划状态与关键入口
2. 中栏（创作主舞台）
   - Chat / Timeline / Storyboard / Clip 四视图
3. 右栏（素材与属性）
   - 分类素材
   - 筛选器
   - 快捷动作

移动端：

1. 左右栏改 Drawer
2. 中栏保持单列主舞台
3. 保留同样的四视图切换

## 4. 导航与菜单

## 4.1 一级导航

1. 项目
2. 创作
3. 素材
4. 风格
5. 历史

## 4.2 二级菜单规则

1. 仅展开当前一级节点。
2. 每个二级菜单最多 6 项，避免滚动式菜单噪音。
3. 工程能力（MCP 管理、技能调试）不进入默认产品路径。

## 5. 视觉系统

## 5.1 色板与 Token

1. Page BG：`#F5F7FA`
2. Card BG：`#FFFFFF`
3. Border：`#E5E7EB`
4. Text Primary：`#111827`
5. Text Secondary：`#6B7280`
6. Brand：`#2563EB`
7. Success：`#10B981`
8. Warning：`#F59E0B`
9. Error：`#EF4444`

## 5.2 组件映射（Ant Design）

1. 布局：`Layout`, `Sider`, `Drawer`, `Splitter`
2. 导航：`Menu`, `Segmented`, `Tabs`, `Breadcrumb`
3. 表单：`Form`, `Input`, `Select`, `Upload`, `Switch`
4. 数据：`Card`, `Collapse`, `List`, `Table`, `Timeline`
5. 反馈：`Alert`, `Message`, `Modal`, `Spin`, `Progress`

## 5.3 动效策略

1. 首屏加载：分区渐进出现（不超过 250ms）
2. 素材新增：卡片高亮闪烁一次
3. 路径切换：主视图平滑切换，不用复杂动画

## 6. 关键用户心流

## 6.1 新手首轮心流（默认）

1. 创建项目
2. 在对话输入一句话启动（系统自动初始化默认序列）
3. 默认风格自动应用（按需再进入风格定制）
4. 选择模式（Checkpoint/YOLO）
5. 生成内容
6. 在 Storyboard 查看结果
7. 在 Clip 视图保存拼接计划

## 6.2 老手复用心流

1. 进入项目
2. 复用历史风格档案
3. 直接发起批量生成
4. 快速筛选与拼接

## 7. 即时引导规格（无步骤卡）

## 7.1 规则

1. 中栏不展示“固定四步步骤卡”。
2. 引导改为上下文即时提示：
   - 无计划时：提示“一句话开始创作”
   - 风格不满意时：提示“按需进入风格定制”
   - 无素材时：提示“继续在对话中推进”
3. Chat 输入区始终是首要视觉焦点。

## 7.2 空态文案

1. 无项目：`还没有项目，先创建一个。`
2. 无序列：`直接输入一句话即可开始，系统会自动初始化工作序列。`
3. 无风格：`先选参考图，系统会自动反推风格。`
4. 无素材：`还没有结果，点击“开始生成”。`

## 7.3 错误文案

必须包含：

1. 出错原因（简短）
2. 建议动作（立即可执行）

## 8. 核心交互细节

## 8.1 模式切换（Checkpoint/YOLO）

1. 顶栏和输入区双重显示当前模式。
2. 切换到 YOLO 时弹一次提示。
3. 新用户默认 `checkpoint`。

## 8.2 Style Init 抽屉

结构：

1. 搜索区
2. 参考图区
3. 反推结果区
4. 已保存 profile 区

动作：

1. 一键应用到当前序列
2. 一键把应用结果注入聊天继续执行

## 8.3 Storyboard 视图

支持：

1. 分类分组
2. 素材预览
3. 快捷动作：
   - 复制 prompt
   - 继续生成
   - 设为风格参考

## 8.4 Clip（最简剪辑）视图

目标：快速形成拼接计划，不做复杂时间线编辑。

功能：

1. 选择视频片段
2. 拖拽排序
3. 每片段设置：
   - in/out（秒）
   - 转场（none/cut/fade）
4. 保存拼接计划（JSON）

## 9. 可用性指标与验收

## 9.1 UX 指标

1. 首次用户 5 分钟内完成首轮生成。
2. 主流程点击步数较现版下降 30%。
3. 空态页面不出现“无指导空白区”。

## 9.2 视觉验收

1. 默认全部主页面为白色体系。
2. 深色仅允许在媒体预览内容中出现，不作为容器基底。
3. 组件圆角、边框、阴影风格一致。

## 9.3 功能验收

1. 主舞台不出现占位式步骤卡，Chat 主控路径清晰。
2. `checkpoint/yolo` 行为差异清晰可见。
3. Storyboard 与 Clip 视图均可完成闭环操作。
4. 移动端可完成完整主流程。

## 10. 实施顺序（开发执行）

1. 统一主题 token 与布局骨架
2. 首次引导与空态
3. Style Init 与 Storyboard 优化
4. Clip 视图与拼接计划
5. 细节交互和移动端修正

   - 一键加入全部视频素材

   - 清晰空态与错误提示

## 12. 引导文案规范

所有引导文案必须满足：

1. 指向明确动作（例如“一句话启动”“去初始化”）。
2. 不出现内部术语（如 tool registry、provider catalog）。
3. 同时说明“为什么做”和“做完会得到什么”。
4. 不超过 2 句，避免段落化说明。

## 13. 组件约束（Ant Design）

1. 菜单与导航只用 `Menu + Segmented + Breadcrumb` 组合。
2. 表单交互只用 `Input/Select/Button/Switch/Upload`，不引入自定义 form 框架。
3. 状态反馈只用 `Alert/Tag/Empty/Spin/Message`，禁止隐式静默失败。
4. 卡片布局优先 `Card`，列表优先 `List`，避免自制复杂容器样式。

## 14. 开源整合策略（强制）

本项目 UI 只采用成熟开源体系，禁止“拼凑式视觉”。

1. 主 UI 框架：`antd`（官方组件）
2. 引导体系：`Steps + Tour`
3. 信息架构：`Layout/Sider/Drawer/Menu/Breadcrumb`
4. 数据承载：`Card/List/Table/Collapse/Timeline`
5. 反馈体系：`Alert/Message/Empty/Spin`

约束：

1. 不新造基础组件（按钮、输入框、弹窗、菜单）；
2. 不使用低对比文本（白底白字、浅灰底浅灰字一律禁止）；
3. 所有关键交互必须有 hover/disabled/loading 三态；
4. 组件文案统一中文，避免中英混杂造成认知断裂。

## 15. 可读性红线（必须满足）

1. 白底容器中的正文文本颜色最低 `#111827`。
2. 次级文案最低 `#6B7280`，且字号不低于 11px。
3. 主流程按钮最小高度 32px；图标按钮需有 tooltip。
4. 移动端保留完整能力，不得“仅展示无操作”。

## 16. v1.3 交互重构（强制）

以下规则覆盖旧版“导航 + 多看板”心智，按本节执行。

### 16.1 左栏定位：控制中枢，不做导航

左栏不再承载一级/二级导航菜单，改为四块：

1. `Agent 实时计划`（最高优先级）
2. `风格状态`（默认风格 / 已切换为自定义）
3. `计划列表`（原 sequence，面向用户统一叫“计划”）
4. `历史会话`

要求：

1. 左栏始终可见当前计划草案或执行中计划。
2. checkpoint 模式下，左栏必须提供“确认计划”入口。

### 16.2 看板收敛

顶部仅保留两个主看板：

1. 对话
2. 剪辑

取消：

1. 时间线独立看板
2. 分镜独立看板

### 16.3 时间线改为条状抽屉

时间线通过素材区按钮打开，展示为条状横向轨道（可滚动），用于快速查看：

1. 当前执行到哪一步
2. 最近工具调用
3. 错误节点

不再作为一个完整页面承载。

### 16.4 素材面板升级（替代分镜看板）

素材面板支持“展开查看”模式（大尺寸）。

展开后每个素材卡必须提供：

1. prompt 可见与复制
2. 基于此素材继续生成
3. 将图片设为风格参考（按需触发风格定制）

### 16.5 剪辑必须可视化

剪辑看板必须提供可视化编辑体验，不要求专业 NLE，但必须具备：

1. 预览区
2. 可视化时间轨（片段块）
3. 片段属性编辑（in/out/转场）
4. 自动粗剪（一键从已有视频素材生成初版）
5. 自动加载已有粗剪结果

禁止将“plan JSON 编辑”作为主交互暴露给普通用户。

### 16.6 计划驱动执行（Checkpoint/YOLO）

1. `checkpoint`：先生成计划草案，用户确认后再执行；执行过程可继续逐步确认。
2. `yolo`：先生成计划草案，然后自动连续执行到结束。
3. 两种模式下，计划都要在左栏可见，避免黑盒执行。

## 17. v1.4 对话主控精修（强制）

### 17.1 首屏重构（`/video`）

目标：进入即创作，不先学习系统。

结构：

1. 中央 `Hero Chat`（大输入框 + 单按钮）
2. 下方 `已创建故事` 列表（最近项目）
3. 顶部仅保留品牌、刷新和最近继续入口

交互：

1. 用户输入一句话后直接创建项目并进入工作台。
2. 若输入为空，禁用创建按钮。
3. 历史故事卡支持一键继续、删除。

文案要求：

1. 输入占位：`说一句你想创作的视频内容，例如：做一个治愈系咖啡品牌短片。`
2. 不出现“序列 / SQ / 初始化文件上传”等术语。

### 17.2 素材操作重构（`ResourcePanel`）

默认素材卡仅保留高频动作：

1. `@该素材`（注入 chatbot 上下文）
2. `设为新风格`（同级按钮，仅图片可用）
3. 点击缩略图打开详情弹窗

详情弹窗结构（强制）：

1. 左侧：大图/视频预览
2. 右侧：`创作描述` 编辑区（可编辑）
3. 右侧底部：`重 roll` 按钮（直接再生成）

约束：

1. 取消素材卡上的“复制 prompt”按钮。
2. 不在卡片上堆叠长文本说明，避免噪音。
3. `@该素材` 注入逻辑与既有“继续生成”一致，但文案统一为 `@该素材`。

### 17.3 提示词保护策略

目标：保护系统内置 prompt，不向玩家暴露。

分层：

1. `系统层`（内置策略、流程 prompt）：不可见
2. `创作层`（用户可理解可编辑描述）：可见

UI 规则：

1. 详情弹窗仅展示创作层描述字段。
2. 任何系统层内容不得在卡片、弹窗、日志里直出。
3. 复制能力仅作用于创作层描述。

### 17.4 左栏重构为 Photoshop 式中枢

左栏固定三块：

1. 计划（顶部卡片，markdown 展示）
2. 风格（当前默认/自定义状态 + Pro 入口）
3. 会话历史（底部，主历史入口）

要求：

1. 不再承担页面导航功能。
2. checkpoint 草案状态必须可见且可确认。
3. yolo 状态需显示“已自动执行”。
4. 左栏底部不重复展示“计划列表”，避免与顶部计划卡语义冲突。

补充（v1.4.1）：

1. 左栏只保留必要信息，不堆叠低频控制。
2. 计划展示以 markdown 原文为主，不拆复杂术语。
3. 任何“技术概念型文案”默认隐藏，避免干扰创作心流。
4. 执行历史从素材面板移至左栏，默认折叠，按需点击展开。
5. 计划区不放风格入口，风格入口独立放在“风格”卡片内。
6. 计划卡支持折叠/展开，并提供简单可视化（步骤完成进度）。

计划卡可视化最小规格：

1. 显示步骤总数与执行步号（例如 `2 / 6`）。
2. 显示阶段状态：`待确认 / 执行中 / 已完成 / 失败`。
3. 支持折叠：折叠态仅显示摘要，不遮挡会话列表。

### 17.5 Pro 模式（高级能力入口）

`Pro` 入口放在左栏风格卡与顶部工具区。

Pro 面板包含：

1. 工作流偏好（路径采纳/屏蔽）
2. 长期记忆（memory user、清理、推荐偏好回显）
3. 高级风格定制（参考图直连生成链路）

约束：

1. 对普通用户默认折叠，不打断主流程。
2. 不暴露 MCP/tool 术语，只给“作用描述 + 操作”。
3. Pro 面板内每项配置都需要“恢复默认”按钮，避免误配后卡死流程。

### 17.6 风格定制真实语义

风格定制不仅是反推词，还必须支持：

1. 将选中图片作为参考图直连图生图/图生视频链路
2. 参考角色（style_ref / scene_ref / character_ref 等）由系统内部处理

UI 对外：

1. 用户只看到“设为新风格”与“已应用到当前计划”。
2. 不展示底层参考参数细节。

### 17.7 剪辑看板强化（可视化可拖动）

最小可用标准：

1. 可视化时间轨块
2. 拖动 in/out 范围（slider/handle）
3. 预览窗口实时反映当前片段
4. 自动粗剪加载 + 可编辑 + 可保存
5. 粗剪自动串播预览（按片段顺序自动播放）
6. 支持拖拽重排片段顺序

禁止：

1. 让用户直接理解或编辑 JSON 才能完成剪辑。

### 17.9 开源剪辑引擎对齐（新增）

目标：剪辑体验向专业开源实现靠拢，而非“表单式伪剪辑”。

参考仓库：

1. OpenCut: [https://github.com/OpenCut-app/OpenCut](https://github.com/OpenCut-app/OpenCut)
2. openreel-video: [https://github.com/Augani/openreel-video](https://github.com/Augani/openreel-video)

落地策略：

1. 先完成能力对齐矩阵（时间线、预览、裁切、重排、导出）。
2. 优先复用可抽取组件或交互模式，不重复造轮子。
3. 先 PoC，再替换当前剪辑台主交互。

成本约束（强制）：

1. 以“最小改动接入”为目标，不进行大规模架构迁移。
2. 若开源仓库无法直接嵌入，则抽取交互范式而非整仓搬运。
3. 首次接入只做 `拖拽重排 + 裁切手柄 + 串播预览` 三件事，避免范围失控。

### 17.8 稳定性验收（强制）

1. 新生成素材必须在 2 秒内进入素材库（正常网络条件）。
2. 生成成功但素材库无结果，视为 P0 缺陷。
3. `docs/api-playbook.md` 补充素材写回链路排查步骤与验证命令。
