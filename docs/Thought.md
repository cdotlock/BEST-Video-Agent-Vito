# Thought

## 我们到底在追求什么

我们追求的不是“更快地产出一个能看的视频”，而是让系统具备接近导演与制片人的判断力：

1. 知道为什么这个镜头要存在。
2. 知道这个镜头和上一个、下一个镜头怎么接。
3. 知道哪些细节是应该被强调的，哪些细节只是噪声。
4. 知道什么时候该更华丽，什么时候该更克制。
5. 知道伟大来自控制，而不是堆砌。

真正的大师级系统，不是 prompt 更长，而是取舍更准。

## 为什么很多 AI 视频系统上不去

它们通常卡在五个地方：

1. 把“好看”误认为“好导演”。
2. 把“信息很多”误认为“镜头很强”。
3. 把“生成成功”误认为“连续性成立”。
4. 把“一个 prompt”误认为“一个 production pipeline”。
5. 把“多工具”误认为“有审美判断”。

于是最后得到的往往是：

- 单帧好看，但镜头之间接不上。
- 角色每个镜头都像不同人。
- 运镜很忙，但没有戏。
- 背景很细，但表演不清楚。
- 分镜很多，但没有 cut value。

## 大师级系统应该像谁思考

它应该同时像四种人思考：

1. 导演
   决定每个镜头的戏剧目的、情绪重心、表演密度。
2. 摄影指导
   决定景别、机位、光线、运动、空间关系。
3. 动画导演
   决定角色是否稳定、姿态是否可读、动作弧线是否干净。
4. 剪辑师
   决定这个镜头有没有进入最终粗剪的资格。

如果系统不能同时承担这四层判断，它就永远只是一个会生成素材的工具，而不是创作系统。

## 为什么当前应聚焦动画短片

因为动画短片是最适合把系统做深、做精、做出差异化的战场。

原因很现实：

1. 动画允许更强的风格控制和世界统一性。
2. 动画比写实视频更依赖设计、分镜和连续性，而这些正是 agent 最能放大的地方。
3. 动画短片的镜头长度、动作密度、资产复用方式，更适合通过 prompt/context/workflow 进行系统化提升。
4. 只要动画短片做透，未来向其他领域做最简支持会容易得多；反过来先铺太广，只会稀释质量。

结论：

`/video` 这条主线应该默认以“动画短片导演工作流”作为第一性原理。

## 伟大作品的共同点

真正让人记住的短片，通常不是因为“生成得很炫”，而是因为以下要素被控制住了：

1. 每个镜头只有一个主意图。
2. 每个镜头都能被剪进去。
3. 角色在不同镜头里仍然是同一个人。
4. 场景不只是背景，而是叙事空间。
5. 空镜、反应镜头、动作镜头之间有呼吸与节奏。
6. 风格统一，但不单调。
7. 镜头服务情绪，而不是抢戏。

因此系统必须从“资产生成器”升级为“镜头组织器”。

## Prompt 不该再只是描述词堆

优秀 prompt 的本质，不是更华丽的修辞，而是更清晰的导演结构。

一个高质量镜头 prompt 至少应隐含六件事：

1. 主体是谁。
2. 这一拍的戏剧动作是什么。
3. 镜头怎么看它。
4. 光线和空间怎么支持它。
5. 它与前后镜头的连续性在哪里。
6. 哪些漂移和噪声必须被压制。

所以 prompt compiler 的职责，不是把一切都拼进去，而是把这些层级编排成稳定的 runtime grammar。

## 大师级 prompt 往往更短，不是更长

很多系统越想变强，prompt 就越写越长。

但真正的问题不在字数，而在密度。

大师级 prompt 往往有三个特征：

1. 每一行都承担明确职责。
2. 几乎没有“解释给模型听”的废话。
3. 只保留真正会改变结果的约束。

所以系统需要的不是继续堆段落，而是一个 `prompt compressor`：

1. 先把导演信息想清楚。
2. 再把它压成执行语言。
3. 把 workflow 元信息、模型元信息、无效形容词从最终生成 prompt 里剥掉。

这一步很关键，因为真正进入图像/视频模型的 prompt，不应该像产品 PRD，更应该像导演在片场给出的短指令。

还有一个经常被忽略的点：

如果是首帧图生视频、首尾帧视频或 mixed refs，文字 prompt 的职责也会变化。

这时最有价值的通常不再是重复描述画面长什么样，而是明确：

1. 这个镜头的目的是什么。
2. 动作如何推进。
3. 镜头如何运动。
4. 节奏如何起、承、落。

也就是说，参考图负责“长相”，文字更多负责“怎么动、为什么动、动到哪里停”。

## 系统设计也不能走向另一种极端

还有一个常见陷阱：

为了提高下限，系统开始堆越来越多预设、越来越多硬编码路径、越来越多固定话术。

这样短期内看似更稳，但会带来三个副作用：

1. agent 不再思考，只是在填空。
2. 新题材、新风格、新工作流一进来就变钝。
3. 上限被提前封死。

所以真正正确的方向不是“少工具、多预设”，而是：

1. 多给工具。
2. 多给原则。
3. 多给结果标准。
4. 多给 recall 能力。
5. 少给死脚本。

我们要的不是把 agent 训练成流水线工人，而是把它训练成可靠的专业合作者。

## 什么叫高下限但不封上限

系统应该像一个很强的制作公司管理机制：

1. 底线明确。
2. 质量标准明确。
3. review 严格。
4. 资产可复用。
5. 但具体创作路径，允许团队因题材而变。

换成 agent 语言就是：

1. 非谈判项必须硬约束。
2. 路径建议只能作为 prior。
3. 工具组合和步骤顺序应动态决定。
4. 只要结果更好，agent 应该允许偏离默认模板。

优秀的系统，不是让 agent “永远照着预设做”，而是让它“很少做蠢事，同时偶尔能做出惊喜”。

## 稳定高质量，比偶发神迹更重要

如果 1 分是满分，很多随便写的 agent 会在 `0.1 - 0.9` 间剧烈摆动。

真正好的系统不是偶尔到 0.95，而是长期稳定在 `0.8 - 0.85`。

这意味着系统设计要优先保证：

1. 不轻易失控。
2. 不反复返工。
3. 不因为预设太死而错失更优路径。
4. 不因为过度自由而频繁掉到低质量区间。

真正的高手，不是每次都赌神抽卡，而是长期稳定交付。

## 动画短片的核心导演原则

### 1. 一镜一意图

短镜头里不要塞太多动作。
一个镜头通常只负责：

- 一个主情绪
- 一个主动作
- 一个主运镜

### 2. 可读性优先于华丽

动画镜头首先要“读得懂”：

- 轮廓是否清晰
- 姿态是否明确
- 主体是否从背景里跳出来
- 观众是否知道应该看哪里

### 3. 连续性高于瞬时惊艳

一个非常漂亮但接不上的镜头，最终是废片。

系统必须天然偏好：

- screen direction 稳定
- eye-line 稳定
- 角色脸型/服装/比例稳定
- 场景地理关系稳定
- 进入和退出动作有逻辑

### 4. 场景不是背景，是叙事容器

好的 scene plate 不是“漂亮空景”，而是可供后续镜头反复进入的舞台。

它必须说明：

- 光源在哪
- 空间怎么分层
- 关键物体在哪
- 人物未来会从哪里进入
- 这个空间适合承载什么情绪

### 5. 动作设计应分层

系统要学会区分：

1. 微表演镜头
   呼吸、眼神、发丝、衣摆、粒子、光影
2. 表演镜头
   转头、抬手、逼近、停顿、反应
3. 动作镜头
   起势、受力、推进、收势

如果这三类动作语法混为一谈，成片必然失控。

## 系统应该怎样自进化

真正值得被记忆和强化的，不是“生成成功了”，而是：

1. 哪种镜头组织方式最容易出片。
2. 用户偏爱的景别和运镜是什么。
3. 用户更喜欢角色优先，还是空镜节奏优先。
4. 哪种风格 token 会导致稳定统一。
5. 哪种 workflow path 经常被废弃。
6. 哪种片段最终总是被留进 clip plan。

系统应该持续学习“成功的原因”，而不是只积累“发生过什么”。

## Review 必须像专业团队

真正有效的 review 不是问“好不好看”，而是问：

1. 这个镜头有没有戏剧目的。
2. 这个镜头能不能剪。
3. 角色有没有漂。
4. 运镜是否抢走了表演。
5. 空镜是不是只在凑数。
6. 动作是不是一条干净弧线，而不是一堆碎动作。
7. 对白镜头里，画面是否给台词留了节奏。

如果 review 不能指向明确改法，它就不是真 review。

## 这次升级的方向

这次不是做大而全，而是提高质量密度：

1. 默认风格基线切到动画短片，而不是通用产品图思维。
2. Prompt Compiler 明确注入导演语法、动画原则、连续性规则和分镜思维。
3. `video_mgr` 在执行前自动补 runtime prompt，而不是完全依赖 agent 临场拼字。
4. style refs 和 active style profile 要在运行时自动进入生成链，而不是靠模型“记得自己去带”。
5. 记忆优化要输出导演约束，而不只是偏好词汇列表。

## 伟大的标准

这套系统什么时候才算开始接近伟大？

不是当它支持了更多按钮，而是当它做到这些：

1. 用户一句话进去，默认就更像一个导演在接任务。
2. 分镜生成开始稳定体现“镜头感”而不是“插画感”。
3. 角色、场景、空镜、动作参考开始真正分工。
4. 视频提示开始更像 shot direction，而不是剧情摘要。
5. 粗剪阶段开始经常出现“可留用镜头”，而不是一堆只能当废片参考的素材。

## 最后一句

奥斯卡级别不是靠一次神 prompt 得到的。
它来自系统长期坚持同一套高标准：

- 对镜头负责
- 对连续性负责
- 对节奏负责
- 对情绪负责
- 对取舍负责

当系统开始承担这些责任，它才不再只是一个会生成的 agent，而是一个真正开始理解电影的人。
